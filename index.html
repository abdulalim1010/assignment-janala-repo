Frequently Asked Questions

the difference between `var`, `let`, and `const`?
Use let and const instead of var to avoid issues related to scope and hoisting. const is typically used for variables whose values won't change, while let is for variables that can be reassigned.
the difference between `map()`, `forEach()`, and `filter()`?
map(): Use when need a modified version of the original array (e.g., transforming data). forEach(): Use when just want to perform an action (e.g., logging or modifying external variables) on each element, but don't need a new array. filter(): Use when need a new array with only the elements that pass a condition (e.g., extracting items that meet certain criteria).
explain arrow functions and how they are different from regular functions?
In general, arrow functions are great for simple, concise functions, especially in callbacks and where this should refer to the surrounding context. Regular functions are better suited for object methods, constructors, and situations requiring access to the arguments object.
how JavaScript Promises work
Promises are a powerful tool for managing asynchronous operations in JavaScript. By using .then(), .catch(), and chaining, you can handle asynchronous tasks in a more readable and maintainable way compared to traditional callback functions. Additionally, Promise.all(), Promise.race(), Promise.allSettled(), and Promise.any() give you advanced options for dealing with multiple asynchronous operations.
how closures work in JavaScript?
A closure is a function that has access to its own scope, the scope of the outer function, and the global scope.
